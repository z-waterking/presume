## 工作经历

### 微软 (Microsoft) | Software Engineering II | 2022.11 - 至今

**项目一：Meta Smart Match (MSM) - Bing 搜索广告智能语义匹配系统**
*担任 Tech Lead (核心技术负责人)*

> **业务背景**：
> Bing 搜索广告系统原有的匹配机制（EEM/EPM）严重依赖于字面或短语层面的精确匹配，难以捕捉长尾查询（Long-tail Queries）背后的真实用户意图，导致系统处于“被动匹配”状态，损失了巨大的潜在商业价值。
>
> **核心职责**：
> 主导设计并落地了 **Meta Smart Match (MSM)** 系统。该系统作为现有匹配链路的强力补充，旨在利用深度语义挖掘技术，主动发现 Query 与广告 Keywords 之间隐含的高价值关联，实现从“字面匹配”到“语义理解”的范式跃迁。

*   **多源异构图挖掘与语义扩展 (Graph-based Mining & Semantic Expansion)**：
    *   构建 **Query-Ad 点击二部图 (Co-click Graph)**，利用 Transitive Join 和 Random Walk 算法挖掘非直观的共现关系。即使 Query 和 Keyword 字面全不相同，只要它们共享相似的广告点击行为，即可被系统捕获，突破了词法匹配的局限。
    *   引入 **离线生成式语义扩展** 流程。结合 Topic Model 和 Query Rewrite 技术，利用预训练模型（PLM）在大规模离线数据上泛化出高置信度的 Query-Keyword 映射表（Look-up Table），大幅降低了在线推理的延迟，实现了“以空间换时间”的高效检索。
    *   通过 **Porter Stemmer** 等 NLP 技术处理词形变体，提升长尾词的归一化能力。

*   **分层排序架构与多目标融合 (Cascading Ranking System)**：
    *   设计了 **L1 粗排 (Scoring)** 与 **L2 精排 (Re-ranking)** 的漏斗架构。
    *   在 L1 阶段，基于历史数据挖掘（Historical Data Mining）构建 **Rank Score**（语义相关性）、**CTR Score**（点击率预估）与 **Bid**（出价）的多目标融合公式。
    *   引入 **Pareto Optimization** 思想，在保证用户体验（Relevance）不下降的前提下，最大化平台的 RPM（Revenue Per Mille）。通过精细化的阈值控制（Threshold Tuning，如文档中的 `Scorethresholds`），精准过滤低质量匹配。

*   **数据闭环与自动化评估**：
    *   搭建 **Historical Bad/Good Case Mining** 流程，自动化收集线上反馈数据反哺模型，形成数据飞轮。
    *   建立了基于 Scope/Aether 的每日/每周自动化数据清洗管道（Data Pipeline），确保特征的时效性和准确性。

> **量化产出 (Key Results)**：
> *   **收入提升**：在北美市场（North America）实现了 **RPM +10%** 的显著增长，这在成熟的广告系统中属于突破性提升。后续迁移至 CN 市场，贡献了额外 **+2% Revenue**。
> *   **奖项荣誉**：凭借极高的技术复杂度和业务影响力，荣获微软内部 **Greatness Award**（通常授予 Senior/Principal 级别）。

### 阿里巴巴 (Alibaba) | 推荐算法工程师 (SDE 2) | AE 技术部 | 2020.07 - 2022.10

**核心职责**：负责 AE 推荐业务中排序及召回模块的全链路迭代升级，助力系统精准匹配用户兴趣。

*   **排序模块：深度控制网络 (Deep Control Network via PID)**
    *   **创新点**：针对推荐策略难以实时调控的痛点，摒弃传统的手工调权，创造性地将 **PID 控制理论** 引入深度学习训练（LTR）。
    *   **实现**：设计实时深度控制模块，根据线上业务目标（期望 AUC/Loss），动态调整各 Label 对应的 Loss 融合权重。通过反馈回路自动寻找当前约束下的最优参数，保证了在线效果的稳定性。
    *   **成果**：取得 **UV 价值 +12%** 的显著提升。

*   **全场域优化：系统解耦与稳定性建设**
    *   对实时流进行了上下游抽像与解耦。在样本侧构建缓存区与缓冲队列，解决了因 Label 回流延迟导致的正负样本比例失衡问题。
    *   利用 Drop Rank 策略选取大模型 Top 10% 的特征用于 LTR 训练，构建了从数据源、样本构造到模型生效的标准化流程。
    *   **成果**：全场域 **IPV +2%**，**UV 价值 +2%**。

*   **召回模块：基于投票理论的个性化融合 (Borda Count Fusion)**
    *   **创新点**：针对多路召回（>10路）合并时的“截断马太效应”（高分源挤压低分但互补源），提出了基于 **Borda 计数法** 的个性化融合方案。
    *   **实现**：利用黑盒优化思想，不依赖绝对分值，而是基于 Item 的相对顺序和召回源权重进行投票排序。离线侧利用生成器动态生成各路权重，在线侧利用评估器指导生成器训练。
    *   **成果**：有效解决了不同召回源分数不可比的问题，取得 **GMV +2.4%** 的效果。

*   **全链路验证：离线评估体系**
    *   搭建 Hitrate 评估流程，对超过 20 种召回源进行统一尺度评估，验证了“粗排打分”定序的有效性。
    *   针对漏斗模式带来的上下游耦合问题，建立专门的评估指标监控曝光占比及转化效率，推动更多新召回源的平滑上线。
